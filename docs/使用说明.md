# PaddleOCR Web UI 使用说明

## 系统概述

基于 PaddleOCR 的 Web 文档识别服务，支持 22 种文件格式，提供 Web 界面和 API 两种使用方式。

---

## 一、支持的文件格式（22 种）

### 直接 OCR 识别（PaddleOCR 原生支持）

| 扩展名 | 中文名称 |
|--------|---------|
| pdf | PDF 文档 |
| jpg / jpeg | JPEG 图片 |
| png | PNG 图片 |
| bmp | BMP 图片 |
| tiff / tif | TIFF 图片 |
| webp | WebP 图片 |

### LibreOffice 转 PDF 后识别

| 扩展名 | 中文名称 |
|--------|---------|
| doc / docx | Word 文档 |
| odt | OpenDocument 文本 |
| rtf | 富文本格式 |
| xls / xlsx | Excel 表格 |
| ods | OpenDocument 电子表格 |
| csv | CSV 表格 |
| ppt / pptx | PowerPoint 演示文稿 |
| odp | OpenDocument 演示文稿 |
| txt | 纯文本 |
| html / htm | HTML 网页 |

### 处理流程

```
图片文件 → PaddleOCR 直接识别
PDF 文件 → PaddleOCR 直接识别
Office 文件 → LibreOffice 转 PDF → PaddleOCR 识别
```

### 文件限制

- 单文件最大：1GB
- 支持多文件同时上传
- 并发数可在管理后台调整

## 二、Web 界面使用

### 2.1 登录

通过 SSO 统一登录，管理员首次需在后台手动设置管理员权限。

### 2.2 上传识别

1. 进入「上传任务」标签页
2. 拖拽或点击上传文件
3. 选择输出格式：Markdown / JSON / 纯文本 / DOCX
4. 点击「开始转换」
5. 等待进度条完成，点击任务查看结果

### 2.3 进度说明

- **图片/PDF**：单进度条 0% → 100%
- **Office 文档**：两阶段进度
  - 「转换PDF」0% → 50%
  - 「OCR识别」50% → 100%

### 2.4 文件管理

- 查看所有已处理文件
- 单文件下载：支持 Markdown / JSON / 纯文本 / DOCX / ZIP 打包
- 批量下载：勾选多个文件 → 打包下载（ZIP 包含源文件 + 图片 + 全部结果）
- 删除：普通用户软删除，管理员可彻底删除

### 2.5 下载内容说明

ZIP 打包下载包含：
```
文件名/
├── source_原始文件名.xxx    # 源文件副本
├── result.md               # Markdown 识别结果（含图片引用）
├── result.json             # 结构化 JSON（按页按块）
├── result.txt              # 纯文本（如有）
├── result.docx             # Word 文档（如有）
├── 原始文件名.pdf           # LibreOffice 转换的 PDF（仅 Office 文档）
└── images/                 # OCR 提取的图片
    ├── img_in_xxx.jpg
    └── ...
```

## 三、API 使用

### 3.1 认证

所有 API 请求需携带 API Key：

```
X-API-Key: ak_xxxxxxxxxxxxx
```

API Key 在管理后台「API Key 管理」中创建。

### 3.2 提交任务

```
POST /api/v1/tasks
Content-Type: multipart/form-data
```

**参数：**

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| file | File | 是 | 要识别的文件 |
| task_type | string | 是 | 固定值 `ocr` |
| output_formats | string | 否 | JSON 数组，如 `["markdown","json"]` |

**示例：**

```bash
curl -X POST http://HOST:5553/api/v1/tasks \
  -H "X-API-Key: YOUR_KEY" \
  -F "file=@document.pdf" \
  -F "task_type=ocr" \
  -F 'output_formats=["markdown","json"]'
```

**返回：**

```json
{
  "task_id": 98,
  "message": "任务已提交"
}
```

### 3.3 查询任务状态

```
GET /api/v1/tasks/{task_id}
```

**返回：**

```json
{
  "task": {
    "id": 98,
    "status": "completed",
    "input_filename": "document.pdf",
    "progress": 100,
    "page_total": 198,
    "processing_time": 150
  },
  "result": "# 识别结果内容..."
}
```

**status 取值：** `queued` → `processing` → `completed` / `failed`

### 3.4 查询任务列表

```
GET /api/v1/tasks
```

### 3.5 下载结果

```
GET /api/v1/files/{task_id}/download?format=FORMAT
```

**format 取值：**

| 值 | 说明 |
|----|------|
| md | Markdown 文件 |
| json | 结构化 JSON |
| txt | 纯文本 |
| docx | Word 文档 |
| zip | ZIP 打包（源文件 + 图片 + 全部结果） |

**示例：**

```bash
# 下载 ZIP 打包（推荐）
curl -O http://HOST:5553/api/v1/files/98/download?format=zip \
  -H "X-API-Key: YOUR_KEY"

# 下载 JSON
curl -O http://HOST:5553/api/v1/files/98/download?format=json \
  -H "X-API-Key: YOUR_KEY"
```

### 3.6 批量下载

```
POST /api/v1/files/batch-download
Content-Type: application/json
```

```json
{
  "file_ids": [98, 99, 100],
  "format": "md"
}
```

返回 ZIP 文件。

### 3.7 取消任务

```
POST /api/v1/tasks/{task_id}/cancel
```

## 四、管理后台

仅管理员可见，包含以下模块：

### 4.1 用户管理

- 查看所有用户
- 设置/取消管理员权限

### 4.2 API Key 管理

- 创建 API Key（仅显示一次完整 Key）
- 吊销 API Key
- 查看完整 Key（内网环境）

### 4.3 系统设置

可在线修改，立即生效，无需重启：

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| 图片 OCR 超时 | 300 秒 | 单张图片识别超时 |
| PDF 每页超时 | 30 秒 | PDF 每页识别超时基准 |
| LibreOffice 转换超时 | 3600 秒 | Office 转 PDF 超时 |
| 健康检查超时 | 10 秒 | OCR 服务健康检查 |
| 图片并发数 | 4 | 图片任务并发上限 |
| PDF 并发数 | 2 | PDF 任务并发上限 |
| 最大文件大小 | 1024 MB | 上传文件大小限制 |

### 4.4 系统日志

- 实时查看系统运行日志
- 按级别着色：ERROR（红色）、WARNING（黄色）、INFO（绿色）
- 支持自动刷新、调整显示行数

## 五、部署

### Docker 部署

```bash
docker build -t paddleocr-ui .
docker run -d -p 5553:5553 \
  -v ./data:/app/data \
  paddleocr-ui
```

Dockerfile 已包含 LibreOffice（writer + calc），支持 Office 格式转换。

### 手动部署

**依赖：**
- Python 3.12+
- LibreOffice（可选，Office 格式需要）
- Node.js（前端构建）

**启动：**

```bash
# 安装 Python 依赖
pip install -r requirements.txt

# 构建前端
cd frontend && npm install && npm run build && cp -r dist/* ../static/

# 启动后端
python -m backend.main
```

**配置：**

可通过 `.env` 文件或环境变量配置，参见 `backend/config.py`。

## 六、技术架构

- **后端**：FastAPI + SQLAlchemy async + PostgreSQL（openGauss-lite）
- **前端**：Vue 3 + Element Plus + Pinia
- **OCR**：PaddleOCR HPS 产线服务
- **Office 转换**：LibreOffice headless
- **实时通信**：WebSocket + HTTP 轮询降级
- **任务队列**：asyncio.PriorityQueue（3 级优先级）
